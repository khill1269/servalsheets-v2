Notable Observations

Schema Weight Issue

sheets_format alone is 241 KB of JSON Schema — nearly 10% of the entire startup payload. The top 5 tools (format, dimensions, advanced, data, visualize) are 912 KB combined. Consider DEFER_SCHEMAS mode to reduce cold-start context window consumption.

Excellent Routing Guidance

The 737-line server instructions include a comprehensive decision tree, 15+ worked examples, and a quick routing matrix. This is unusually thorough for an MCP server and should significantly reduce tool selection errors.

Rich Resource Ecosystem

68 resources + 12 templates provide on-demand knowledge (guides, patterns, examples, decision trees, schema details) without bloating the initial payload. The knowledge:///search resource enables fuzzy search across all documentation.

Missing: Tool Icons

Icons are declared in serverInfo (server-level icon present) but individual tool-level icons arrays are empty (0 per tool). The code generates 25 SVG icons but they may not be wired into tools/list output.

0-Action Schemas

sheets_collaborate and sheets_federation show 0 action variants in oneOf/anyOf. Their schemas may use a different discriminated union structure (flat z.object vs nested oneOf), which could impact Claude's ability to discover valid actions.

Webhook Actions Hidden

sheets_webhook shows only 4 of 10 actions. The other 6 (register, unregister, list, get, test, get_stats) are hidden because Redis is not connected — this is the enrichInputSchema() filter working correctly.
