All 17 Code Bugs — Fix Priority

Priority 1: A1 Notation Pipeline (4 bugs, 1 root cause)

BUG-2: data.find_replace — Invalid A1 from range resolution
Handler generates 'Sheet'!1:3 (row-only) which fails A1 validation. Root cause: sheetId → range resolution outputs row-only notation.
BUG-6: format.auto_fit — Same A1 resolution issue
Same root cause as BUG-2. Single fix in the range resolution utility would fix both.
BUG-7: format.generate_conditional_format — NaN sheetId from range
Range 'Budget vs Actuals'!A1:F10 resolves sheetId to NaN. Sheet names with spaces break the parser.
BUG-14: advanced.update_named_range — Stored range validation
Existing named ranges with row-only notation (stored by Google) fail re-validation on update. Need to relax A1 validation for existing ranges.

Priority 2: Output Schema Mismatches (4 bugs)

BUG-1: core.get — Output validation: sheet.index undefined
Google API doesn't return index field. Fix: make index optional in output schema.
BUG-8: advanced.named_functions — suggestedFix object vs string
Error response has fixableVia as object, schema expects string. Blocks all named function CRUD. Fix: update error response type.
BUG-12: compute.batch_compute — percentiles not iterable
options.percentiles is not iterable for statistical type. Fix: add default percentiles: [25, 50, 75].
BUG-15: agent.execute_step — stepId vs stepIndex mismatch
Schema expects stepId: string, plan responses use numeric indices. Fix: accept both or align response format.

Priority 3: Google API Parameter Bugs (3 bugs)

BUG-4: visualize.chart_move/chart_resize — overlay_position snake_case
Sends overlay_position (snake_case) to Google API which expects overlayPosition (camelCase). Fix: rename field.
BUG-5: appsscript.list_processes — Invalid query param
scriptProcessFilter.scriptId not accepted as query param. Fix: use correct API path format.
BUG-13: advanced.update_banding — Missing fields parameter
Google API requires fields mask. Fix: add fields: '*' or specific field list.

Priority 4: Schema/Discriminator Bugs (4 bugs)

BUG-3: data.smart_fill — fillRange discriminated union
Natural params (range, sourceRange, strategy) don't satisfy the schema. Fix: simplify discriminated union.
BUG-16: analyze.execute_plan — Plan schema too rigid
Plan requires order, type, priority, estimatedDuration, reason, outputs. Fix: make most optional.
BUG-17: analyze.drill_down — Invalid target discriminator
Target object schema requires specific type discriminator that doesn't match expected values. Fix: update discriminated union.
BUG-11: quality.analyze_impact — Missing operation param
Requires operation object that isn't documented. Fix: document or make optional.

Priority 5: DuckDB Engine Bug (2 actions affected)

BUG-10: compute.sql_query / sql_join — DuckDB read_json_auto IO error
read_json_auto() treats JSON data as file path. Fix: use read_json_auto('/dev/stdin') or write temp file. File: src/services/duckdb-engine.ts
