Confirmed Bugs (15)

BUG-1: sheets_core.get — Output validation: sheet.index undefined
The get action's output schema expects index: number for each sheet, but Google API doesn't return index in the response payload.
BUG-2: sheets_data.find_replace — Invalid A1 notation generated internally
Handler generates 'Sheet'!1:3 which fails A1 validation. The handler internally constructs an invalid range reference from sheet metadata.
BUG-3: sheets_data.smart_fill — Missing fillRange discriminated union
Schema requires fillRange as primary field, but natural params (range, sourceRange, strategy) don't satisfy the discriminated union.
BUG-4: sheets_visualize.chart_move / chart_resize — Invalid field: overlay_position
Handler sends overlay_position (snake_case) to Google API which expects overlayPosition (camelCase). Affects both chart_move and chart_resize.
BUG-5: sheets_appsscript.list_processes — Invalid query parameter
Handler sends scriptProcessFilter.scriptId as a query parameter, but the Apps Script API doesn't accept it in that format.
BUG-6: sheets_format.auto_fit — Invalid A1 notation from range resolution
Handler generates row-only notation like 'SheetName'!1:3 which fails A1 validation when resolving sheetId to range.
BUG-7: sheets_format.generate_conditional_format — NaN sheetId from range
When given a range like 'Budget vs Actuals'!A1:F10, the handler resolves the sheetId to NaN. Range-to-sheetId resolution fails for sheet names with spaces.
BUG-8: sheets_advanced.named_functions — Output validation error
list_named_functions and create_named_function return error with suggestedFix as an object when output schema expects string. Affects all named function CRUD.
BUG-9: sheets_compute.python_eval / pandas_profile — Pyodide sandbox blocks compile()
compile() is blocked in the WASM sandbox. Also affects matplotlib_chart and sklearn_model. All Python-based compute actions fail.
BUG-10: sheets_compute.sql_query/sql_join — DuckDB read_json_auto IO error
DuckDB's read_json_auto() treats the JSON data string as a file path instead of parsing inline data. Code bug in src/services/duckdb-engine.ts.
BUG-11: sheets_quality.validate — Requires undocumented value param
Schema validation requires a value field (union type) that is not documented. spreadsheetId + range alone are insufficient.
BUG-12: sheets_compute.batch_compute statistical — percentiles not iterable
"options.percentiles is not iterable" when running statistical type in batch_compute. Missing default percentiles initialization.
BUG-13: sheets_advanced.update_banding — Missing fields parameter
Google API 400 error. Handler does not set required fields parameter for the updateBanding request. API requires explicit field list or * wildcard.
BUG-14: sheets_advanced.update_named_range — A1 validation on stored range data
Named ranges with row-only notation (e.g., 'Scenario - Q2'!1:3) stored in the spreadsheet cause validation failure when attempting to update any named range.
BUG-15: sheets_agent.execute_step — Schema expects stepId not stepIndex
The execute_step action schema expects stepId: string but documentation and plan responses reference steps by numeric index. Schema-response mismatch.
