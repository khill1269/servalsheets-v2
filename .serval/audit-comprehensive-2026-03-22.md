## 🔴 Wave 2 Critical

### CI/CD Supply Chain

| ID | File:Line | Issue |
|---|---|---|
| W-C1 | `.github/workflows/auto-draft-pr.yml:67` | Commit message injected via `${{ steps.commits.outputs.summary }}` into JS template literal — RCE in GitHub Actions runner |
| W-C2 | `.github/workflows/docker.yml:82` | `trivy-action@master` pinned to mutable branch with `attestations: write` — supply chain compromise via unpinned action |

**W-C1 Detail:** The auto-draft-pr workflow interpolates the raw commit summary (which includes commit messages from all contributors) directly into a JavaScript Actions expression. An attacker who can create a commit with a specially-crafted message can execute arbitrary code in the Actions runner with `attestations: write` and repo-write access. Fix: use `toJSON()` or pass via env variable.

**W-C2 Detail:** `trivy-action@master` is a mutable ref. Any commit to that repo's main branch runs with `attestations: write` permission. Additionally, the CI security scan step has `continue-on-error: true`, making the entire dependency CVE gate a no-op. The `node-forge@0.10.0` transitive dep (via node-saml→xml-encryption) has multiple CVSS ≥8.0 CVEs.

### Sandbox Escapes

| ID | File:Line | Issue |

| W-C8 | `src/handlers/compute.ts:1307, 1332, 1491` | Python code injection via template literal interpolation — `targetColumn`, `xColumn`, `title` values inserted directly into sklearn/matplotlib Python source code without sanitization |
| W-C9 | `src/analysis/conversational-helpers.ts:239-254` | Prompt injection — sheet names, column headers, cell values, and prior AI response history injected raw (no delimiters, no sanitization) into the `systemPrompt` field sent to the sampling server |
